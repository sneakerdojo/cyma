1. Design Decisions

AspectDecisionRationale
Interaction modelAgent-driven guided flowAgent decides sequence — feels like a wizard but adapts to each user
UI appearanceSame as current wizardReuse proven components (OctoChoices, OctoTextInput patterns)
Previous stepsHidden — "History" toggleClean focus on current step; history available on demand with Edit
OrbCurrent size, pinned topBrand presence without stealing space from interaction area
Thinking stateOrb pulses + dots + "thinking..."Clear feedback that something is happening
Text fallbackIntegrated into each componentNo duplicate bottom bar — one clean component zone per step
Voice inputMic button on text fallbackSpeak to answer any question — transcribed + sent as text
Detail textAlways visibleContext helps users give better answers
Agent outputMarkdown + Mermaid diagramsRich responses — agent can explain with visuals, not just text
Diagram displayInline, tap-to-expandReadable inline; expand for complex diagrams

8. Data Flow

8.1 User → Agent

1
User interacts with component

Taps a choice, checks boxes, types text, records voice, uploads file, or submits form

2
Previous step collapses

Current step stored in local history[] array with { question, answer, toolId, toolInput }

3
Orb → thinking state

Faster pulse, glow intensifies, "thinking..." label appears

4
Message sent to agent

sendMessage({ text }) via useChat → POST /chat/stream

5
Agent processes + responds

Calls data tools (enrich_lead, answer_service_question), writes text, calls UI tool

6
Stream complete → render

Orb → speaking. Question typewriters in. Detail fades. Component slides up. Text fallback appears.

8.2 Voice Input Flow

1
User taps mic button

MediaRecorder starts (reuses existing code from OctoTextInput.tsx)

2
User speaks → stops recording

Audio blob created (webm on Chrome, mp4 on iOS Safari — dynamic MIME detection already built)

3
Transcription

Audio sent to backend → transcribed via Whisper API or Kimi audio model → text returned

4
Text sent as answer

Transcribed text sent as sendMessage({ text }) — agent sees it as a normal text response

8.3 History + Edit Flow

1
Each completed step stored locally

{ stepId, question, answer, toolId, toolInput, timestamp }

2
User taps "History" button

Drawer slides open showing all past answers as compact pills with "Edit" links

3
User taps "Edit" on an answer

The original component re-renders with the previous value pre-filled

4
User changes their answer

New value replaces old in history. Sent to agent as: [Edited: {field} changed from "{old}" to "{new}"]
